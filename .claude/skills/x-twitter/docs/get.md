Retrieves one or more posts by ID. Pass a single ID or comma-separated IDs. Maps to GET /2/tweets/:id (single) or GET /2/tweets (multiple, up to 100). Invoke via `node <base_directory>/x.js get <id> [flags]` or `node <base_directory>/x.js get <id1,id2,...> [flags]`. Output is JSON to stdout.

[!FLAGS] a) no flags — returns the post(s) with default tweet fields (text, author_id, created_at, conversation_id, public_metrics, article, attachments) and expands author (name, username) and media. Posts with images, videos, or GIFs will have an `attachments.media` array containing the resolved media objects (url, type, width, height, alt_text; use `preview_image_url` for video/GIF thumbnails). Article tweets (long-form posts) will include `article.title`, `article.plainText`, `article.previewText`, and `article.mediaEntities` automatically. b) `--fields <comma-list>` — override default tweet fields (if you override, include `article` to avoid missing article content). c) `--raw` — output the full API response envelope (data, includes, errors).

[!OUTPUT-SHAPE] Single ID: produces the post object directly. Multiple IDs: produces an array of post objects. Posts with attached media have `attachments.media_keys` (the raw keys) and `attachments.media` (resolved objects with URLs). With `--raw`, wraps into the API envelope with `data` and `includes` (expanded users and media).
